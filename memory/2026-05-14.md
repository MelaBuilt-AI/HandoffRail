## Key decisions
- SDK uses Pydantic v2 models (matching server) but standalone — no runtime dependency on server package
- Namespace package pattern (`handoffrail/sdk/`) allows coexistence with server
- from_dict/to_dict on every model for easy serialization
- Both sync (httpx.Client) and async (httpx.AsyncClient) clients
- Retry with exponential backoff on connection errors
- Custom exception mapping from HTTP status codes to typed SDK exceptions
